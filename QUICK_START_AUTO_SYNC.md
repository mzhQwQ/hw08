## ❓ 常见问题

### Q1: 登录失败或跳转不了

**原因**：可能需要 VPN 或账号密码错误

**解决方案**：
```
1. 检查学号和密码是否正确
2. 确保有网络连接
3. 如在国外，需要连接 VPN
4. 如有双因素认证，完成认证步骤
5. 清除浏览器 Cookie 后重试
```

---

### Q2: JS 脚本执行失败

**症状**：控制台显示 `❌ 未找到选择框`

**原因**：页面结构与脚本预期不同

**解决方案**：
```bash
# 调试步骤：
1. 在控制台执行：
   console.log(document.querySelector('select[name="year"]'))
   
2. 如返回 null，说明选择框不存在
   - 这可能表示页面格式不同
   - 可以跳过 JS 脚本，手动选择学年学期
   
3. 或者从『学年』和『学期』下拉框手动选择
   - 选择『全部』→ 点击查询
   - 等待页面加载
```

---

### Q3: HTML 粘贴后显示"无法找到成绩表"

**原因**：复制的 HTML 不完整或格式不对

**解决方案**：
```
1. 检查复制的内容
   - 在编辑器中粘贴看看是否有内容
   - 应该包含 <html> 和 <table> 标签
   
2. 重新完整复制
   - 按 Ctrl+A 全选（不要手动框选）
   - 按 Ctrl+C 复制
   
3. 确保页面完全加载
   - 等待 2-3 秒让页面加载完成
   - 看到成绩表格中有课程数据
   
4. 尝试用浏览器开发者工具复制
   - 按 F12 打开开发者工具
   - Elements 标签 → 右键 <html> 标签
   - 选择 Copy → Copy outerHTML
```

---

### Q4: 解析成功但成绩为 0 或显示不正确

**原因**：从绩点反推成绩不够精确

**说明**：
```
这是正常的。应用从绩点（如 3.67）反推成绩分数（如 84）。
由于反推算法的必然偏差，可能与实际分数有 ±5 分左右的差异。

但这不影响诊断结论，因为：
✓ 诊断是基于绩点比较，不是绝对分数
✓ 相对排名是准确的
✓ 优势和弱点分析仍然正确

如需精确分数，可以：
1. 在原始成绩页面查看具体分数
2. 检查教务系统的成绩单
```

---

### Q5: 程序运行缓慢或经常卡住

**原因**：网络问题或服务器响应慢

**解决方案**：
```bash
# 方案 1：等待片刻后重试
# 高峰时段（如课程结束后）教务系统访问量大

# 方案 3：查看应用日志

# 方案 4：清除缓存
rm score_cache.json
streamlit cache clear
streamlit run app.py
```
